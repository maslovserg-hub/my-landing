## 2. СТРУКТУРА ПРОДАЮЩЕГО ЛЕНДИНГА

### Принцип: «Путь доверия»

```
Захват внимания → Понимание ценности → Доказательство → Детали → Действие
```

70% пользователей покидают страницу за 15 секунд, если не понимают оффер. Каждый блок отвечает на конкретный вопрос в голове посетителя.

---

### БЛОК 1 — HERO (Первый экран)
**Вопрос клиента:** «Кто это и что он делает для меня?»

**Содержимое:**
- Имя + специализация (конкретная)
- Заголовок-оффер с результатом для клиента
- Подзаголовок — расшифровка (макс. 2 строки)
- Фото/аватар (человеческий контакт)
- Одна главная CTA-кнопка
- Вторичная ссылка («Смотреть работы»)
- Счётчики: проекты / опыт / показатели

**Лучшие практики:**
- Заголовок = результат, не звание: «Создаю сайты и ИИ-агентов, которые приносят клиентов»
- CTA: «Обсудить проект» или «Получить консультацию» — не «Связаться»
- Фоновый элемент: анимация кода / градиент / particle system — без перегруза

**Пример:**
```
[Имя Фамилия]
Разработчик сайтов, приложений и ИИ-агентов

Превращаю идеи в рабочие продукты —
от лендинга до автономного ИИ-ассистента

[Обсудить проект]  [Посмотреть работы ↓]

47 проектов  ·  5 лет опыта  ·  Срок — всегда
```

---

### БЛОК 2 — УСЛУГИ / ЧТО Я ДЕЛАЮ
**Вопрос клиента:** «Подходит ли это мне?»

**Содержимое:**
- 3–4 карточки с иконками
- Название от клиента (не от специалиста)
- Результат, не процесс + срок/стоимость «от»
- Микро-CTA на каждой карточке

**Пример карточек:**

| Услуга | Описание | Срок |
|--------|----------|------|
| Сайт под ключ | Лендинг или корпоративный сайт. Дизайн + разработка + SEO | от 7 дней |
| Мобильное приложение | iOS/Android. От прототипа до публикации в сторах | от 3 недель |
| ИИ-агент для бизнеса | Чат-бот, автоответчик, агент обработки заявок | от 5 дней |
| Автоматизация | CRM + мессенджеры + ИИ. Меньше ручного — больше продаж | по задаче |

---

### БЛОК 3 — ЦИФРЫ / СОЦИАЛЬНОЕ ДОКАЗАТЕЛЬСТВО
**Вопрос клиента:** «Можно ли этому человеку доверять?»

**Содержимое:**
- 3–5 крупных метрик с подписями
- Логотипы компаний-клиентов (повышают конверсию на 22%)
- Краткое кредо

**Правило:** Конкретные цифры («47») работают лучше округлённых («50+»)

```
47          34%         3 дня        98%
проектов    средний     быстрее      клиентов
сдано       рост        дедлайна     рекомендуют
            конверсии
```

---

### БЛОК 4 — ПОРТФОЛИО / КЕЙСЫ
**Вопрос клиента:** «Он умеет делать именно то, что мне нужно?»

**Содержимое:**
- 4–6 проектов: превью + название + результат
- Фильтры по типу (сайты / приложения / ИИ)
- Кнопка «Смотреть все работы»

**Структура карточки:**
```
[Превью проекта]
Категория: ИИ-агент
Название: Бот-продавец для стоматологии
Результат: +67% записей через мессенджеры
[Смотреть кейс →]
```

**Правило:** Проблема → решение → результат в цифрах. Живые ссылки на проекты убеждают в 40% случаев сильнее, чем статичные скриншоты.

---

### БЛОК 5 — КАК Я РАБОТАЮ / ПРОЦЕСС
**Вопрос клиента:** «Что произойдёт после того, как я напишу?»

**Содержимое:**
- 4–5 шагов с номерами
- Название + описание + что получает клиент на этом этапе
- Гарантии на каждом шаге

**Пример:**
```
1. Созвон 30 мин
   Рассказываете задачу — я задаю нужные вопросы.
   Никаких брифов на 10 страниц.

2. ТЗ и смета
   Фиксируем объём, стоимость, сроки.
   Всё письменно — никаких сюрпризов.

3. Разработка с показом
   Работаю итерациями, показываю каждые 2–3 дня.

4. Тестирование и правки
   Проверяем на всех устройствах. 5 попыток правок включены.

5. Запуск и поддержка
   Передаю доступы. Первый месяц поддержки — бесплатно.
```

---

### БЛОК 6 — ОТЗЫВЫ / TESTIMONIALS
**Вопрос клиента:** «Другие уже рискнули и не пожалели?»

**Содержимое:**
- 3–6 карточек: фото + имя + должность + компания + конкретный текст
- Результат в цифрах обязателен
- Видеоотзыв: конвертирует в 2–3 раза лучше текста

**Пример сильного отзыва:**
```
"Обратился за лендингом для курса. Андрей сам предложил
добавить квиз — в итоге конверсия выросла до 8,3%.
Такого я не ожидал."

Михаил Орлов, CEO онлайн-школы «Деньги в порядке»
```

### БЛОК 9 — О СЕБЕ / ЛИЧНЫЙ БРЕНД
**Вопрос клиента:** «Это живой человек или фабрика?»

**Структура:**
1. Opening hook (боль клиента или неожиданный факт)
2. Специализация + позиционирование
3. Конкретные цифры экспертизы
4. Личная нотка-дифференциатор
5. Переход к действию

**Избегать:** «опытный», «ответственный», «клиентоориентированный»

---

### БЛОК 10 — ФИНАЛЬНЫЙ CTA + КОНТАКТЫ
**Цель:** Убрать всё трение между желанием и действием

**Содержимое:**
- Заголовок-триггер (не «Контакты»)
- Форма: максимум 3 поля
- Telegram-кнопка (конвертирует в 2–3× лучше email для рунета)
- Время ответа: «Отвечаю в течение 2 часов»
- Микро-гарантия: «Без навязывания и обязательств»

**Пример:**
```
Есть проект? Обсудим бесплатно за 30 минут

[Имя]
[Telegram / WhatsApp]
[Задача в двух словах]

[Отправить заявку]

или сразу: @username — отвечаю до 3 часов
```

### Тренд 1 — Scroll-driven Animations (нативные CSS-анимации на скролл)

**Что это:** Анимации, привязанные к позиции скролла без JavaScript. CSS Scroll-driven Animations поддерживаются Chrome 115+, Edge 115+, Firefox 110+.

**Применение:**
- Hero-текст «вылетает» при скролле вниз
- Карточки кейсов появляются с fade-in + slide-up
- Прогресс-бар чтения в хедере
- Parallax на фоновые элементы

**Примеры:** linear.app, stripe.com, vercel.com

```css
@keyframes fadeSlideUp {
  from { opacity: 0; translate: 0 40px; }
  to   { opacity: 1; translate: 0 0; }
}

.card {
  animation: fadeSlideUp linear both;
  animation-timeline: view();
  animation-range: entry 0% entry 40%;
}
```

**Сложность:** Низкая (CSS) / Средняя (GSAP ScrollTrigger)

---

### Тренд 2 — Кастомный курсор + Magnetic buttons

**Что это:** Замена системного курсора на брендированный blob с задержкой (lerp), меняющий форму/текст при hover на элементы.

**Применение:**
- Маленькая точка (4px) + follower-кружок 40px с задержкой
- На карточках проектов: курсор → «VIEW» / «OPEN»
- Magnetic buttons — кнопка тянется к курсору при приближении

**Примеры:** bruno-simon.com, camillemormal.com, tiborsagain.com

```javascript
const cursor = document.querySelector('.cursor-dot');
const follower = document.querySelector('.cursor-follower');
let mouseX = 0, mouseY = 0, followerX = 0, followerY = 0;

document.addEventListener('mousemove', e => {
  mouseX = e.clientX; mouseY = e.clientY;
  cursor.style.transform = `translate(${mouseX}px, ${mouseY}px)`;
});

function animateFollower() {
  followerX += (mouseX - followerX) * 0.08; // lerp
  followerY += (mouseY - followerY) * 0.08;
  follower.style.transform = `translate(${followerX}px, ${followerY}px)`;
  requestAnimationFrame(animateFollower);
}
animateFollower();

// Magnetic effect
document.querySelectorAll('.btn-magnetic').forEach(btn => {
  btn.addEventListener('mousemove', e => {
    const rect = btn.getBoundingClientRect();
    const x = (e.clientX - rect.left - rect.width / 2) * 0.3;
    const y = (e.clientY - rect.top - rect.height / 2) * 0.3;
    btn.style.transform = `translate(${x}px, ${y}px)`;
  });
  btn.addEventListener('mouseleave', () => btn.style.transform = '');
});
```

**Сложность:** Средняя

### Тренд 3 — Dark Mode + Neon Accent Colors

**Что это:** Тёмный фон (#0A0A0F, не чисто чёрный) с яркими неоновыми акцентами и glow-эффектами.

**Цветовые схемы:**
- **AI/Tech:** `#0A0A0F` + `#00D4FF` (electric blue) + `#BF5AF2` (neon purple)
- **Профи:** `#0a192f` + `#64ffda` (Brittany Chiang — эталон)
- **Техно-матрица:** `#000000` + `#00FF88`
- **Минимал:** `#111111` + `#FFFFFF` + один акцент

**Примеры:** raycast.com, resend.com, planetscale.com

```css
:root {
  --bg: #0A0A0F;
  --accent: #00D4FF;
  --accent2: #BF5AF2;
  --glow: 0 0 20px rgba(0,212,255,.4), 0 0 60px rgba(0,212,255,.15);
}

.hero-title {
  background: linear-gradient(135deg, #00D4FF 0%, #BF5AF2 50%, #FF6B6B 100%);
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
}

.card:hover {
  box-shadow: var(--glow);
  border-color: rgba(0,212,255,.5);
}

/* Ambient glow blob */
.hero::before {
  content: '';
  position: absolute;
  width: 600px; height: 600px;
  background: radial-gradient(circle, rgba(0,212,255,.1) 0%, transparent 70%);
  border-radius: 50%;
  top: -200px; left: -200px;
  filter: blur(40px);
}
```

**Сложность:** Низкая

### Тренд 4 — Glassmorphism 2.0

**Что это:** Эволюция glassmorphism — тонкие стёкла с noise-текстурой, цветными оттенками, правильным контекстом.

**Применение:**
- Sticky nav-bar
- Карточки кейсов поверх blurred background
- Теги/бейджи со скиллами

**Примеры:** apple.com (Vision Pro), craft.do, Awwwards SOTD 2024–2025

```css
.glass-card {
```

### Тренд 5 — Kinetic Typography (кинетическая типографика)

**Что это:** Текст как главный визуальный элемент — scramble, marquee, variable fonts, split-text. Буквы живут и двигаются.

**Применение:**
- Text scramble при загрузке («DEVELOPER» → случайные символы → «AI ENGINEER»)
- Бегущая строка со скиллами / логотипами клиентов
- Variable font: weight меняется при hover

**Примеры:** studiomoniker.com, hugeinc.com, waaark.com

```javascript
// Text Scramble
class TextScramble {
  constructor(el) {
    this.el = el;
    this.chars = '!<>-_\\/[]{}—=+*^?#@ABCDEFabcdef0123456789';
  }
  setText(newText) {
    const length = Math.max(this.el.innerText.length, newText.length);
    return new Promise(resolve => {
      this.queue = Array.from({ length }, (_, i) => ({
        to: newText[i] || '',
        start: Math.floor(Math.random() * 20),
        end: Math.floor(Math.random() * 20) + 10,
      }));
      this.frame = 0; this.resolve = resolve;
      this.update();
    });
  }
  update() {
    let output = '', complete = 0;
    for (const { to, start, end } of this.queue) {
      if (this.frame >= end) { complete++; output += to; }
      else if (this.frame >= start)
        output += `<span style="opacity:.4">${this.chars[Math.floor(Math.random()*this.chars.length)]}</span>`;
      else output += '';
    }
    this.el.innerHTML = output;
    if (complete === this.queue.length) this.resolve();
    else { requestAnimationFrame(() => this.update()); this.frame++; }
  }
}
```

### Тренд 7 — 3D / WebGL элементы (Three.js / Spline)

**Что это:** Интерактивные 3D-объекты в браузере: вращающаяся сфера, particle system, fluid simulation. Spline позволяет без WebGL-знаний.

**Применение:**

- Particle sphere как фоновая анимация
- Объект реагирует на движение мыши

**Примеры:** bruno-simon.com, active-theory.com

```javascript
import * as THREE from 'three';
const scene = new THREE.Scene();
const camera = new THREE.PerspectiveCamera(75, innerWidth/innerHeight, 0.1, 1000);
const renderer = new THREE.WebGLRenderer({ canvas: document.querySelector('#hero-canvas'), alpha: true });
renderer.setSize(innerWidth, innerHeight);

// Particle sphere
const geo = new THREE.BufferGeometry();
const positions = new Float32Array(3000 * 3);
for (let i = 0; i < 3000; i++) {
  const theta = Math.random() * Math.PI * 2;
  const phi = Math.acos(2 * Math.random() - 1);
  const r = 2 + Math.random() * 0.2;
  positions[i*3]   = r * Math.sin(phi) * Math.cos(theta);
  positions[i*3+1] = r * Math.sin(phi) * Math.sin(theta);
  positions[i*3+2] = r * Math.cos(phi);
}
geo.setAttribute('position', new THREE.BufferAttribute(positions, 3));
const mesh = new THREE.Points(geo, new THREE.PointsMaterial({ size: 0.02, color: 0x00D4FF }));
scene.add(mesh);
camera.position.z = 5;

let mouseX = 0, mouseY = 0;
document.addEventListener('mousemove', e => {
  mouseX = (e.clientX/innerWidth - 0.5) * 2;
  mouseY = (e.clientY/innerHeight - 0.5) * 2;
});

(function animate() {
  requestAnimationFrame(animate);
  mesh.rotation.y += 0.002 + mouseX * 0.001;
  mesh.rotation.x += mouseY * 0.001;
  renderer.render(scene, camera);
})();
```

#### Типичные ошибки новичков в структуре

**1. Нет чёткого оффера на первом экране**
Самая частая ошибка — hero говорит КТО ты, а не ЧТО ПОЛУЧИТ клиент. «Разработчик с опытом 5 лет» — это биография. «Запускаю AI-агентов, которые экономят вашим сотрудникам 20 часов в месяц» — это оффер.

**2. Слишком много услуг без иерархии**
Три направления (сайты + приложения + AI) — это риск размытия фокуса. Новички перечисляют всё подряд без визуальной иерархии. Клиент не понимает, на чём специализация. Решение: выдели одну ключевую услугу визуально, остальные — вторично.

**3. Портфолио без контекста**
Скриншоты без описания задачи, результата и роли автора — мёртвый груз. Пользователь смотрит на красивую картинку и не понимает, зачем это ему.

**4. Форма с 5+ полями**
Каждое лишнее поле снижает конверсию на 11%. Имя + мессенджер + задача — достаточно. Остальное — на созвоне.

**5. Нет sticky-навигации с CTA**
Пользователь проскроллил до середины, заинтересовался — а кнопки «написать» нет рядом. Всегда держи CTA в зоне досягаемости.

**6. Мобильная версия — второй сорт**
60–70% трафика на портфолио приходит с телефона (особенно из Telegram, ВКонтакте). Проектируй mobile-first.

#### Что ОБЯЗАТЕЛЬНО должно быть

| Элемент | Почему критично |
|---------|----------------|
| Фото автора в hero | +34% доверия — люди покупают у людей |
| Один ключевой CTA на каждом экране | Без этого скролл не конвертирует |
| Реальные скриншоты проектов | Мокапы на MacBook — клише, которое уже не работает |
| Конкретные цифры в кейсах | «+67% конверсия» убеждает, «красивый сайт» — нет |
| Время ответа рядом с формой | Снимает тревогу ожидания |
| Якорная навигация по секциям | Пользователь должен знать, где он находится |

#### Что раздражает пользователей и заставляет уйти

- **Автовоспроизведение видео или звука** — немедленный закрытый таб
- **Попап с подпиской в первые 3 секунды** — до того как человек понял, зачем он здесь
- **Медленная загрузка** (>3 сек) — уходит 40% пользователей. Особенно критично с анимациями: GSAP + Three.js без lazy load = катастрофа на мобиле
- **Горизонтальный скролл на мобиле** — если не реализован правильно, ломает опыт
- **Текст на тёмном фоне с плохим контрастом** — glassmorphism красив, но читаемость обязательна (WCAG AA минимум)
- **Анимации, которые нельзя пропустить** — если intro-анимация длится >2 сек без skip, 30% уходят
- **Отсутствие «живых» ссылок на проекты** — если нельзя проверить работу прямо сейчас, доверие падает

#### UX-рекомендации специфически для AI-портфолио

- **Добавь демо прямо на странице** — если один из агентов можно попробовать в iframe или виджете, это убьёт конкурентов. Живое демо = лучший кейс
- **Визуализируй сложное просто** — схема «до/после автоматизации» (было: 3 часа вручную → стало: 5 минут) понятнее любого технического описания
- **Не пугай технологиями** — LangChain, RAG, vector embeddings не говорят ничего бизнес-аудитории. Переводи на язык результата

### 📊 ЭКСПЕРТ 2 — Маркетолог, специалист по конверсии

#### Оффер, который зацепит клиента

Сильный оффер строится по формуле:
```
[Конкретный результат] + [для кого] + [за какое время] + [без чего]
```

**Слабо:** «Разрабатываю сайты, приложения и AI-агентов под ключ»

**Сильно:** «Запускаю AI-автоматизацию для малого бизнеса за 2 недели — без найма разработчиков и IT-отдела»

**Главная проблема трёх направлений в одном лендинге:**
Три аудитории (нужен сайт / нужно приложение / нужна автоматизация) имеют разные боли. Универсальный лендинг конвертирует хуже нишевого на 30–50%. Варианты решения:
- Один лендинг с чётким **первичным оффером** (например, AI-агенты как флагман) + остальные услуги вторично
- Или три отдельных лендинга с одним общим hub-портфолио

**Для максимальной конверсии в 2026 — рекомендую AI-агентов как якорный оффер:** высокий средний чек ($5–15k), низкая конкуренция, растущий спрос (+109% YoY), высокий wow-эффект для демонстрации.

#### Триггеры доверия — по приоритету

**Уровень 1 — критичные (без них конверсия близка к нулю):**
1. Реальное фото + имя (не аватар, не иллюстрация)
2. Кейсы с измеримым результатом (не «сделал сайт», а «конверсия выросла с 0.8% до 3.4%»)
3. Отзыв с именем, должностью и компанией клиента

**Уровень 2 — усилители:**
4. Число завершённых проектов (конкретное, не «50+»)
5. Логотипы клиентов или известных партнёров
6. Ссылки на живые проекты
7. Гарантия или условие возврата

**Уровень 3 — дифференциаторы:**
8. Видеоотзыв (конвертирует в 2–3× лучше текста)
9. Публичность: статьи, выступления, GitHub-активность
10. Рейтинг на платформе (Upwork, Avito Услуги)

#### Путь посетителя от «зашёл» до «оставил заявку»

```
ОСОЗНАНИЕ (0–5 сек)
Hero → «Это про меня? Он делает то, что мне нужно?»
Задача: задержать. Оффер + визуал должны дать ответ немедленно.
↓
ИНТЕРЕС (5–30 сек)
Услуги + цифры → «А он вообще хорош? Сколько он сделал?»
Задача: сформировать первичное доверие. Цифры + логотипы.
↓
ОЦЕНКА (30 сек – 3 мин)
Портфолио + кейсы → «Это похоже на мою задачу?»
Задача: показать релевантный опыт. Хотя бы один кейс должен попасть в точку.
↓
УБЕЖДЕНИЕ (3–5 мин)
Отзывы + процесс → «Он надёжный? Что будет после оплаты?»
Задача: снять страх. Процесс работы + гарантии + реальные люди.
↓
РЕШЕНИЕ (финал)
CTA → «Хочу, но как начать? Это ни к чему не обязывает?»
Задача: убрать последнее трение. Лёгкий первый шаг + микро-гарантия.
```

**Ключевой инсайт:** 80% решений принимается на первых двух экранах. Если hero + услуги не зацепили — портфолио уже не поможет.

#### Конкретные конверсионные механики

- **Лид-магнит:** «Бесплатный аудит автоматизации — найду 3 процесса, которые можно автоматизировать прямо сейчас» → конверсия в платный проект 30–40%
- **Дефицит:** «Беру 2–3 проекта в месяц. Сейчас свободно 1 место» — ограниченная доступность повышает срочность
- **Якорение цен:** показывай дорогой пакет первым — средний кажется разумным на его фоне
- **Micro-commitment:** «Ответьте на 3 вопроса → получите предварительную смету» — низкий порог входа

### 👤 ЭКСПЕРТ 3 — Потенциальный клиент, владелец малого бизнеса

*Внутренний монолог при посещении портфолио*

#### Первые 5 секунд — что он видит и думает

> «Окей, кто это? Что он вообще делает? Это для меня или нет?»

**Смотрит на:**
- Заголовок — если не понял с первых слов, закрывает
- Фото — есть живой человек или нет? Нет фото = нет доверия
- Кнопку — есть ли простой первый шаг?

**Его фильтр:** «Он работает с такими как я?» — Размытая специализация («делаю всё») вызывает недоверие. Нишевый («автоматизирую продажи для интернет-магазинов») — сразу понятно.

#### Что заставит ОСТАТЬСЯ

1. **Кейс, похожий на его ситуацию** — «О, он делал для стоматологии — у меня тоже клиника, интересно»
2. **Конкретный результат в цифрах** — «+67% записей» говорит больше, чем абзац текста
3. **Понятный процесс работы** — «Просто пишу и он разберётся? Или мне нужно готовить бриф на 20 страниц?»
4. **Цена или хотя бы ориентир** — отсутствие цен тревожит («а вдруг это очень дорого и я потрачу время зря?»)
5. **Человечность** — живой язык, не корпоративный. «Пишу код под джаз» запомнится лучше, чем «опытный специалист»

#### Что заставит ЗАКРЫТЬ ВКЛАДКУ

1. **Не понял оффер за 5 секунд** — уходит к следующему в поиске
2. **Нет ни одного кейса из его сферы** — «наверное, не моё»
3. **Сложный технический язык без перевода** — LangChain, RAG, vector store — это не про него
4. **Нет цен и ни одного ориентира** — «не хочу тратить время на звонок ради того, чтобы узнать, что это мне не по карману»
5. **Форма просит имя, email, телефон, описание, бюджет, сроки** — перегруз, закрывает
6. **Сайт медленно грузится** — терпение бизнесмена минимально
7. **Нет ответа на вопрос «а вдруг обманет?»** — нет гарантий, нет договора, нет поэтапной оплаты

#### Его главный невысказанный вопрос

> «Окупится ли это? Сколько я заработаю / сэкономлю, если заплачу этому человеку?»

Каждый элемент лендинга должен отвечать на этот вопрос. ROI — главный аргумент для бизнеса.

### ✅ СВОДНЫЙ СПИСОК РЕКОМЕНДАЦИЙ

#### Критично — без этого лендинг не работает

- [ ] H1 = конкретный результат для клиента, не описание себя
- [ ] Фото живого человека в hero
- [ ] Один главный CTA-оффер (не «Связаться», а «Получить бесплатный аудит»)
- [ ] Минимум 3 кейса с цифрами результата
- [ ] Отзыв с именем + должность + компания
- [ ] Telegram как основной канал (для рунета)
- [ ] Форма: максимум 3 поля
- [ ] Мобильная версия работает идеально

#### Важно — повышает конверсию в 1.5–2×

- [ ] Лид-магнит: бесплатный аудит/консультация как первый шаг
- [ ] Счётчики проектов/лет/часов сэкономленных клиентам
- [ ] Логотипы клиентов или скриншоты из реальных мессенджеров (с разрешения)
- [ ] Блок «Как я работаю» — снимает страх неизвестности
- [ ] Ценовые ориентиры или пакеты — хотя бы «от N рублей»
- [ ] Время ответа рядом с формой («отвечаю за 2 часа»)
- [ ] Sticky-хедер с CTA-кнопкой

#### Дифференциаторы — выделяет на фоне конкурентов

- [ ] Демо AI-агента прямо на странице (виджет/iframe)
- [ ] Схема «до/после» для каждого типа услуги
- [ ] Видеоотзыв хотя бы один
- [ ] Публичность: ссылка на GitHub, Telegram-канал, статьи
- [ ] Дефицит: «осталось 1 место в апреле»
- [ ] Личный факт-дифференциатор в секции «О себе»

#### Ошибки — избегать любой ценой

- [ ] Автозапуск видео/звука
- [ ] Попап в первые 10 секунд
- [ ] Анимации тяжелее 100кб без lazy load
- [ ] Текст с низким контрастом на тёмном фоне
- [ ] Intro-анимация длиннее 1.5 сек без skip
- [ ] Технический жаргон без перевода на язык результата
- [ ] «Мы» вместо «я» на личном лендинге — дистанцирует

#### Специфика для AI-специалиста

- [ ] Позиционируй AI-агентов как флагман — самый высокий чек и wow-эффект
- [ ] Оффер через экономию: «N часов в месяц» или «окупается за 6 недель»
- [ ] Никаких «LangChain/RAG/vector DB» без объяснения — только «умный бот, который отвечает вместо менеджера»
- [ ] Живое демо агента = самый сильный кейс
- [ ] Три услуги в одном лендинге — ок, но одна должна быть якорной визуально
