# Правила работы с проектом

## Структура файлов

- Все страницы делаем в одном HTML-файле — никаких отдельных страниц
- Никаких внешних библиотек (Bootstrap, jQuery и т.д.) — только чистый HTML, CSS и JS
- CSS и JS пишем прямо в `<style>` и `<script>` внутри HTML-файла

## Дизайн

- Современный, чистый, минималистичный стиль
- Приятная цветовая палитра — без кислотных цветов
- Достаточные отступы, воздух между блоками
- Кнопки и интерактивные элементы должны выглядеть кликабельными

## Адаптивность

- Всё должно хорошо выглядеть на мобильных устройствах (с 320px и шире)
- Используем `flexbox` или `grid` вместо фиксированных размеров
- Шрифты и отступы адаптируем через `clamp()` или медиазапросы
- Мобильная версия — приоритет, не послесловие

## Стиль текста

- Простой и дружелюбный язык — как будто объясняешь другу
- Без канцелярита: не «осуществить», а «сделать»; не «в рамках», а «для»
- Короткие предложения, конкретные слова
- Заголовки — ёмкие, без воды

## Перед созданием нового файла

- Сначала объяснить, что планируется создать и зачем
- Дождаться подтверждения от пользователя
- Только потом писать код

## Команда завершения дня: "На сегодня всё"

Когда произносишь эту команду, я должен:

1. **Закоммитить изменения** — добавить все незакоммиченные файлы в git и создать коммит с описанием
2. **Проверить документацию** — убедиться, что README.md и другие docs файлы актуальны
3. **Обновить документацию** — если нужны изменения в документации, обновить её
4. **Запушить в remote** — отправить все коммиты в origin

Коммит-сообщение должно кратко описывать все сделанные за день изменения в формате: 
`Day wrap-up: описание основных изменений`
