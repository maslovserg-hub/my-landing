## 1. ЦЕЛЬ САЙТА И ЦЕЛЕВАЯ АУДИТОРИЯ

### Цель
Конвертировать холодного посетителя в горячую заявку через путь доверия: захват внимания → понимание ценности → доказательство → устранение страха → действие.

**Главная метрика успеха:** количество заявок (заполненных форм + переходов в Telegram).
**Целевая конверсия:** 3–5% от уникальных посетителей.

### Позиционирование
**Якорный оффер (флагман):** ИИ-агенты и автоматизация — самый высокий чек, растущий спрос (+109% YoY), низкая конкуренция.
**Дополнительные направления:** сайты, мобильные приложения.

**Главное УТП:** «Один специалист закрывает три задачи: сайт → приложение → ИИ-автоматизация» — редкая комбинация, которую используем как главный дифференциатор.

### Целевая аудитория

| Сегмент | Боль | Что ищет |
|---------|------|----------|
| Владелец малого бизнеса | Менеджеры тратят 3+ часа на ручной ввод | Автоматизировать рутину, окупить за 6 недель |
| Предприниматель без IT | «Хотим сайт/приложение, но боимся дорогой команды» | Один надёжный человек под ключ |
| Стартап / онлайн-школа | Сайт есть, но заявок нет | Лендинг, который конвертирует |
| Малый бизнес (клиники, магазины) | Отвечают на одинаковые вопросы 50 раз в день | ИИ-бот, который заменит менеджера |

**Ключевой невысказанный вопрос клиента:** «Окупится ли это? Сколько я заработаю / сэкономлю?»
Каждый элемент лендинга отвечает на этот вопрос через ROI.

### Анимации (приоритет — Высокий)

| Элемент | Анимация | Технология | Сложность |
|---------|----------|-----------|-----------|
| Hero заголовок | Text Scramble при загрузке (random chars → текст) | Vanilla JS | Средняя |

| Карточки | fade-in + slide-up на scroll | CSS Scroll-driven / GSAP | Низкая |

| Фон hero | Aurora blob медленное движение | CSS анимация | Низкая |
| Particle sphere | Three.js / tsParticles, реакция на мышь | Three.js | Высокая |

### Анимации (приоритет — Средний)

| Кнопки | Magnetic effect при hover | Vanilla JS |

- Изображения: WebP + lazy loading
- Шрифты: `font-display: swap`

## 7. МОБИЛЬНАЯ АДАПТАЦИЯ

60–70% трафика на портфолио приходит с мобильных (особенно из Telegram).
Проектируем **mobile-first**.

**Критичные правила:**
- Горизонтальный скролл кейсов — нативный scroll-snap (без GSAP на мобиле)
- Кастомный курсор — **отключить** на touch-устройствах
- Three.js particle sphere — fallback: статичный CSS-gradient blob
- Кнопки — минимум 44×44px touch target
- Текст на тёмном фоне — проверить контраст WCAG AA (4.5:1 минимум)
- Sticky CTA-хедер — присутствует всегда
- Telegram-кнопка — крупная, в нижней части экрана
